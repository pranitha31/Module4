Q1. Role of Frontend (FE)
User Interface (UI): Designs the look and feel (HTML, CSS, JS).
User Interaction: Handles clicks, forms, navigation, animations.
Communication with Backend: Sends/receives data via APIs (e.g., REST, GraphQL).

Q2. Role of Backend (BE)
Server-side Processing: Executes logic, handles requests, generates responses.
Database Handling: Stores, retrieves, and manages data securely.
Security & Authentication: Validates users, protects data, enforces rules.

Q3. Business Logic
Definition: Rules and workflows that define how data is processed to meet business goals.
Examples:
E-commerce: Apply discounts, calculate shipping costs.
Banking: Validate transactions, calculate interest.
Healthcare: Check patient eligibility, schedule appointments.

Q4. Client–Server Model
Client: User’s device/browser requesting services.
Server: Machine providing resources, data, or services.
Communication: Happens via requests (HTTP) and responses (data, files).

Q5. Three-Tier Architecture
Presentation Layer: UI for users (frontend).
Application/Business Layer: Processes rules and logic.
Data Layer: Database storing information.
Why Used: Improves scalability, security, and separation of concerns.

Q6. JavaScript as a Backend Language
Performance: Non-blocking, event-driven (Node.js).
Ecosystem: Huge library support (npm).
Popular Frameworks: Express.js, NestJS, Next.js..
